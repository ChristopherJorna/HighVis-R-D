HighVis R&D · Programme Hub · 2026-04-28

Direct the shot in Unreal. Let AI handle the photoreal.

HighVis is a tool-agnostic infrastructure for restyling low-fidelity Unreal renders into photoreal video, while preserving camera, composition, and motion. We're building it three ways in parallel. Each one is a fork: same architecture, different model stack, different infrastructure trade-off. We then compare the forks on the same scenes with the same questions, so the answer still holds when the models change.

Local · workflow shape
Preview / Final toggle on every run
Stage 1 always renders a forced-720p preview MP4 (~30-45s). A run_final toggle gates Stage 2: ON adds the source-resolution MP4 plus the 16-bit EXR sequence; OFF stops after preview. MP4 saves before EXR inside each stage so a bad preview is killable before the EXR write. Implemented via ImpactSwitch on the latent feeding the final sampler — no input means ComfyUI skips the entire downstream chain.
Local · reproducibility
Frame rate + count auto-extracted from source
Output fps and frame count come from GetVideoComponents.frame_rate on the input clip — single source of truth wired to SaveVideo, EXR writer, and EmptyLTXVLatentVideo. Two override toggles (use_manual_fps, use_manual_frames) expose manual widgets when needed. Frame count snaps to LTX-valid 8N+1 via SimpleMath+. Means a 30 fps source comes out at 30 fps without anyone touching a number.
Local · GPU access
RunPod mounted as a local drive
Cloud GPU treated as Z:\ via rclone + WinFsp SFTP mount of the pod's /workspace. Compute jobs go through ssh runpod "...". Keeps this Claude session, its memory, and its agents in place — the pod is just disk + GPU. Network Volume is the persistence layer (survives pod stop/destroy). Eleven-step bootstrap checklist lives at state/handover-runpod-highvis-local.md.
